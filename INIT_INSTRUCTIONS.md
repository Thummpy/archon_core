# Initialization Instructions

Step-by-step guide for creating a new project from this template. **Delete this file after initialization is complete.**

## Overview

This template uses a brainstorm-then-initialize workflow:

1. Create a repo from this template
2. Brainstorm your project on claude.ai — it generates a `project_seed.md` file
3. Place the seed in `.claude/project_seed.md` and run `/init` in Claude Code
4. Run `/bootstrap` to create GitHub Issues and plan initial tasks

The result is a fully configured AI-assisted development environment with your project's architecture, standards, and initial tasks ready to go.

## Step 1 — Create Your Repository

Use GitHub's "Use this template" button to create a new repository. Clone it locally:

```bash
git clone <your-new-repo-url>
cd <your-repo-name>
```

## Step 2 — Brainstorm Your Project

Open a conversation on [claude.ai](https://claude.ai) and brainstorm what you want to build. This is not a questionnaire — it's a real technical discussion. Cover:

- **Purpose** — What does this project do? Who uses it?
- **Tech stack** — Languages, frameworks, databases, infrastructure
- **Architecture** — Components, data flow, deployment model
- **Roadmap** — What gets built first? What are the phases?
- **Constraints** — Security requirements, compliance needs, performance targets

When the conversation has covered enough ground, ask claude.ai to generate a `project_seed.md` file. Download it.

Take your time here. The quality of your initialization depends on the clarity of these decisions.

## Step 3 — Place the Seed

Replace the placeholder content in `.claude/project_seed.md` with the seed generated by claude.ai:

```bash
# Overwrite the placeholder with your generated seed
cp ~/Downloads/project_seed.md .claude/project_seed.md
```

## Step 4 — Initialize with Claude Code

Open Claude Code in your project directory and run:

```
/init
```

This reads `.claude/project_seed.md` and populates every template file — `.claude/CLAUDE.md`, `.claude/PLANNING.md`, `README.md`, docs, scripts, CI config, code examples, and more. All `{{PLACEHOLDER}}` markers are replaced with project-specific content derived from the seed.

## Step 5 — Validate

The `/init` command runs validation automatically, but you can also run it manually:

```bash
./.claude/scripts/init-project.sh
```

Fix any reported errors and re-run until all checks pass.

## Step 6 — Bootstrap

Once validation passes, run:

```
/bootstrap
```

This reads your populated project files, then:

- Plans infrastructure setup tasks
- Plans initial development tasks based on your roadmap
- Creates GitHub Issues with labels, descriptions, and phased milestones

**Prerequisite:** The `gh` CLI must be installed and authenticated (`gh auth status`).

## Step 7 — Clean Up and Commit

```bash
# First commit
git add -A
git commit -m "feat: initialize project from ai-dev-framework template"
```

## Step 8 — Configure Repository Settings

- Enable branch protection on `main` (require PR reviews, require status checks)
- Enable CODEOWNERS enforcement so framework file changes require lead review
- Verify the CI workflow (`.github/workflows/validate.yml`) runs on PRs

## Step 9 — Cleanup

```bash
# Delete this file — it's no longer needed
rm INIT_INSTRUCTIONS.md
```
