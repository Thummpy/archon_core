# AI Dev Framework

A GitHub template repository providing a structured AI-assisted development framework built on the WISC methodology (Write, Isolate, Select, Compress) for Claude Code.

**New here?** Start with [INIT_INSTRUCTIONS.md](INIT_INSTRUCTIONS.md) to initialize your project from this template.

## How It Works

The framework uses a **three-tier progressive context system** to give the AI agent exactly the information it needs — no more, no less:

| Tier | Location | Loading | Purpose |
|------|----------|---------|---------|
| 1 | `.claude/CLAUDE.md` | Always loaded | Project identity, universal standards, task protocol |
| 2 | `.claude/rules/` | Auto-loaded by file path | Domain-specific conventions (testing, infra, docs) |
| 3 | `.claude/docs/` | On-demand via sub-agents | Heavy reference material (API docs, compliance guides) |

## Slash Commands

| Command | Description |
|---------|-------------|
| `/init` | Read `.claude/project_seed.md` and populate all template placeholder files |
| `/research` | Iterative pre-plan research to resolve unknowns before planning |
| `/plan-feature` | Research codebase and generate a PRP (implementation plan) for a feature |
| `/execute` | Read a PRP and implement it step-by-step with validation |
| `/review` | Self-review changes against standards, rules, and security checklist |
| `/commit` | Generate and execute a conventional commit with WHY-focused message |
| `/handoff` | Capture session progress to `.claude/HANDOFF.md` for the next session |
| `/compact` | Write handoff and prepare context for compaction in long sessions |
| `/bootstrap` | After init, plan infrastructure and create initial GitHub Issues |
| `/init-subsystem` | Generate a focused context-loading command for a subsystem |

## Repository Structure

```
├── INIT_INSTRUCTIONS.md               # Setup guide (delete after init)
├── README.md                          # Project README
│
├── .claude/
│   ├── CLAUDE.md                      # Tier 1: always-loaded AI context
│   ├── PLANNING.md                    # Architecture, tech stack, constraints
│   ├── HANDOFF.md                     # Session continuity (generated by /handoff)
│   ├── settings.local.json            # Claude Code permission allowlist
│   ├── commands/                      # Slash commands (WISC lifecycle)
│   ├── rules/                         # Tier 2: auto-loaded by file path
│   ├── docs/                          # Tier 3: on-demand reference + project docs
│   │   ├── architecture.md            # System architecture documentation
│   │   ├── setup.md                   # Development environment setup
│   │   ├── deployment.md              # Deployment and operations guide
│   │   └── adr/                       # Architecture Decision Records
│   ├── examples/
│   │   ├── patterns/                  # Annotated code pattern examples
│   │   └── reference-implementations/ # Complete working examples (added during init)
│   ├── prps/
│   │   ├── templates/                 # PRP templates (base, bugfix, refactor)
│   │   └── completed/                 # Archive of executed PRPs
│   └── scripts/
│       ├── validate.sh                # Lint, test, build validation suite
│       └── init-project.sh            # Post-init placeholder checker
│
└── .github/
    ├── CODEOWNERS                     # Framework file governance
    ├── pull_request_template.md       # PR checklist and structure
    └── workflows/validate.yml         # CI validation on PR
```

## File Types

Files in this template are one of three types:

- **Generic** — Works for any project as-is. Universal standards and conventions.
- **Placeholder** — Contains `{{PLACEHOLDER}}` markers with instructions. Replaced during initialization with project-specific content.
- **Scaffolded** — Ships empty or minimal. Populated during active development.
