# Documentation Guide Conventions

## Structure

- Every doc in `docs/` must start with a **"What you need before starting"** prerequisites section.
- Every doc must end with a **"Something went wrong?"** section linking to `TROUBLESHOOTING.md`.
- Docs assume zero Docker knowledge — explain what each command does and why, not just what to type.
- Include "what you should see" expected output after each step.

## Style

- Use step-by-step numbered lists for procedures.
- Use code blocks with copy-pasteable commands.
- Keep paragraphs short — 2-3 sentences max.
